Commit message:
Add create stable onboarding

This patch changes login behavior:
- If a logged-in user has no stable_id, they see a Welcome / Create Your Stable page.
- They can create a stable, add their name, choose admin/trainer/staff role, and then enter the app.
- Existing users with stable_id still go straight to their stable.

No SQL is required unless the app shows a row-level security error while creating the stable.
If that happens, run:
RUN_IF_ONBOARDING_RLS_FAILS.sql
in Supabase SQL Editor.
